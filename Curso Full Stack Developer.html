HTML

Descrição

É uma linguagem para identificar por meio de etiquetas os elementos de uma página web. Com ela é configurada a estrutura dos elementos de uma página, como: Título (Title), Cabeçalho (Head), Corpo (body). Quando são atribuidas etiquetas a um texto cada uma dessas partes se torna um objeto. Com isso passam a ser estruturados em um "Documento Object Model"(DOM) (clique aqui para mais informações sobre o DOM).

Quando o texto é estruturado em varios objetos seguindo a lógica do DOM, ele se torna passível de ser trabalhado com outras linguagens voltadas aos objetos como JavaScript, por exemplo.

Para criar um HTML recomenda-se criar uma pasta para poder inserir o conteúdo criado e as alterações armazenadas em sistemas de versionamento de códigos (Version Control System - VCS), tal como o Git.

É convencionado nomear o arquivo HTML como "Index" por que é um padrão de busca para leitura deste tipo de arquivo. Subentende-se que qualquer página tem o "Index" e quando é nomeado desta foram fica escondida esta expressão no campo do navegador. Até pode ser usado outro nome e se for colocado ira aparecer por extenso no navegador, mas é comum ser denominado de "Index".

Na criação de um novo HTML uma dica é usar o simbolo de "!" e a tecla Tab. Os elemetos do HTML são apresentados com uma tag inicial e uma tag final. Recomenda-se que o conteúdo das tags seja disposto com indentação (não é obrigatório para HTML, mas outras linguagens exigem a indentação) para facilitar a leitura (Atalho SHIT+ALT+F). A indentação consiste em colocar o conteúdo deslocado para a direita com tabs para que o leitor consiga distinguir entre o que foi inserido em cada elemento do HTML, e para casoso em que há subdivisões de sessões (elementos inseridos dentro de outros elementos). É possivel deslocar trechos do codigo selecionando esse trecho epresionando "ALT" e as setas pra cima ou pra baixo.

Existem também elementos HTML que não possuem ambos os comandos de início e fim, possuem apenas um único comando. Esses são chamados de elementos vazios. Para que as linhas do código não fiquem muito longas pode ser selecionado o trecho onde isso acontece no VSC Studio e com o botão direito escolher formatar o documento.

Tags básicas do HTML

As tags básicas gerais do HTML são:

  1. "!doctype";
  2. "html";
  3. "head";
  4. "body";
  5. "h1"*, "h2", "h.."; e
  6. "p"

* - O "H1" trata do titulo geral da página, recomnda-se que cada página tenha apenas um titulo.

Todas as tags apresentam atributos para definições próprias e para atribuir alguma caracteristica de identificação ou de estilo. Os navegadores modernos interpretam já as tags com estruturas já pré-estabelecidos. Porém, essa forma de estruturar o HTML pode não ser aquela desejada pelo desenvolvedor. Para evitar armadilhas o programador pode optar por fazer um trabalho mais minusioso, e definir padrões que ele deseja (idioma,tipos de caracteres que ira utilizar, etc.). A tag "!doctype" determina qual é a linguagem na qual está estruturado um documento. O "html" determina o espaço dentro do qual está escrito o html. Dentro deste tag é possível definir atribuos como: a) a linguagem da página e, com isso, as ferramentas de busca poderão dar preferencia para um público que emprega esse idioma; b) elementos de estilo CSS. Dentro da tag "head", ou cabeçalho podem ser definidos aspectos como: o conteúdo da aba de navegação com o uso da tag "title", e a forma de representação de caractéres (no caso do portugues recomenda-se o uso do UTF-8 que inclui particularidas e como o "til" e a cedilha") com o uso da tag "meta". Além disso, empregando a tag meta podem ser definidos outros atributos da página como compatibilidade com navegadores, dispositivos, responsibidade.

GIT

GIT é um sistema de código aberto para controle de versões de arquivos (Version Control System - VCS). A página oficial para efetuar o download deste sistema é git-scm.com Dentro da página tem videos, livros e outros materiais que tratam sobre o sistema (incluindo um livro em portugues). Dentro do sistema GIT tem varias ferramentas: GUI, CMD e BASH, por exemplo. O GIT GUI é uma interfase gráfica. É aconselhavel usar o GIT BASH por que é um terminal que contem as linhas de comando. Por trabalhar com linhas de comando é aconselhével para trabalhar com outros sistemas que trabalham dessa forma. Dentro do terminal do GIT o responsável por processar os comando do servidor é o SHELL. Shells são aplicativos terminais usados como interface em sistemas operacionais por meio de comandos gravados. Com ele é possível reverter arquivos para uma versão anterior, comparar alterações, ver as versões ao longo do tempo, quem fez uma modificação que incorreu em algum erro. Com isso, no lugar de ter diversas copias de um mesmo arquivo, é possivel ter apenas um arquivo em uma pasta e trabalhar suas versões a partir do git. É interessante que o arquivo que o usuário queira versionar seja salvo em uma pasta. Ao ser inserido no Git, o arquivo gerara um arquivo oculto denominado ".git". Este arquivo é importante por que contem todas alterações e versões armazenadas no git. Muitas pessoas usam copias dos arquivos que desejam versionar copiando eles para um novo diretório com a identificação temporal (Time Stamped Directory)

O diretório com identificação temporal é alterado quando o próprio diretório é alterado. O diretório contém, entre outras coisas, uma lista dos inodes dos arquivos no diretório, portanto, quando você alterar o conteúdo do diretório adicionando ou removendo arquivos, o carimbo de data e hora dos diretórios será atualizado.

Git se destaca entre as modalidades de VCSs por permitir o armazenamento de versões na nuvem - Github. Em suma, com o GIT a partir de um arquivo original (Arquivo mestre ou arvore original) é possível controlar alterações inserindo novos layouts, elementos denominados de branchs (galhos). Além disso, o Git é uma ferramenta que permite trabalhar simultaneamente com outros desenvolvedores. Todas as modificações feitas são identificadas. O Git pode ser inserido dentro do VSCode para tornar prático o controle de versões do arquivo. Para isso, deve ser aberto um novo terminal e clicar no sinal de adição e escolher GIT BASH

Comandos do terminal Git BASH

Antes de efetuar qualquer alteração o usuário precisa criar uma identidade. Para isso deve empregar dois comandos:

  • Comando "git config --global user.email "e-mail do usuário".
  • Comando "git config --global user.name "Nome do usuário".

Outros comando utilizados no git são:

  • Comando "ls": apresenta todos os documentos do local onde está posicionado o sistema.
  • Comando "Clear" (atalho: Ctrl+L): Limpa todos os comandos inseridos no terminal.
  • Comando "cd" (Change Directory): Acessar um diretório do computador.
  • Comando "cd .." (Change Directory): Volta para o diretório anterior.
  • Comando "git init" = Cria versão original ou mestre de um arquivo (Na pasta é possivel ver esta versão visualizando os arquivos ocultos da mesma).
  • Comando "git add" = adiciona uma versão com alterações da versão mestre.
  • Pode ser informado o arquivo por extenso ou pode ser adicionado tudo com "."
  • Comando "git status" = apresenta todas as versões do git e sobre qual estamos trabalhando em destaque (verde)
  • Comando "git commit" = Submete as últimas alterações feitas no codigo fonte. Para acrescentar uma descrição que explique a alteração feita, pode ser acrescentado "-m" (m-message) e informar a alteração feita nessa versão.
  • Comando "git Log" = presenta as inforamções sobre a versão do código e da identificação de quem fez as alterações.
  • Comando "git branch" = lista as versões do arquivo.
  • Comando "checkout" = cria uma nova branch adicionando '-b' (branch) e o nome da mesma ou acessa o conteúdo de uma das versões existentes, colocando o nome da mesma.
  • Comando "git merge" = incorpora as versões alteradas à versão mestre
  • Comando "git push" = serve para salvar o projeto em um repositório/nuvem. A primeira vez em que este código é salvo na nuvem tem que ser inserido o termo "-u" e o nome do arquvo main atribuido ao codigo mestre.
  • Comando "git clone (endereço onde está localizado o reositório git)" = cria uma versão local do código armazenado na nuvem.

Observação: se ocorrer qualquer erro de digitação na navegação entre diretórios pode acontecer como não ecerrar bem a descrição da pasta com a aspa (precisa haver apenas duas aspas: uma para abrir e uma segunda para fechar a descrição) ao pressionar "Enter" o terminal não entende o comando e vai para uma linha nova iniciada com o sinal ">" nesse caso é preciso encerrar a expressão com uma aspa final simples e acionar o "Enter" para voltar ao formato onde aparece o $ que permite inserir um novo comando.

Uma sequência de comandos que pode ser utilizada ao abrir o terminal pela primeira vez seria: a) configurar o usuário (nome e e-mail), b) explorar os arquivos da pasta onde o terminal está aberto com "ls", c) buscar a pasta onde esta o projeto, d) acessar a pasta com o comando cd e o nome da pasta até o local onde esta o código, e) criar um arquivo git dentro da pasta usando o código 'git init', f) verificar o arquivo com 'git status' e 'git log', para sair da visualização do 'git log' clicar em 'q' (quit) g) após alterar o arquivo (acrescentar ou retirar conteudo), verificar se precisam ser adicionadas alterações com 'git status' e analisar a mensagem, h) adicionar uma versão com 'git add' e o nome do arquivo ou . (para trabalhar nesta mesma versão, 'master' por exemplo), i) Se for feita uma alteração significativa pode ser criada uma nova branch utilizanco o comando 'git checkout -b' e nomeando essa branch descrevendo o que foi alterado na versão mestre. j) descrever as alterações feitas com "git commit -m", k) salvar o projeto seguindo os passos informados na mesma:

Quando se trata de um novo projeto:
echo "# Curso-Full-Stack-Developer" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/Juan-Debali/Curso-Full-Stack-Developer.git
git push -u origin main.

Projeto já existente:
git remote add origin https://github.com/Juan-Debali/Curso-Full-Stack-Developer.git
git branch -M main
git push -u origin main

CSS

CSS é uma linguagem utilizada para estilizar um arquivo HTML. Ele serve para descrever a forma como os elementos HTML serão apresentados. Para definir o estilo de um HTML existem tres formas:

  1. Inserindo a tag "Style" no elemento que deseja ser estilizado.
  2. Colocando a tag do "Style" dentro da tag "head" e escolher como estilizar uma etiqueta empregada no corpo do HTML.
  3. A forma recomendada é criar um arquivo novo com terminação ".css" (de preferência denominado "style.css") e referenciar esse arquivo com a tag "link" dentro da tag "head" do arquivo HTML. Uma dica é criar uma pasta de "Estilos" e inserir o estilos desejados dentro desta pasta.

A linguagem CSS é atribuida a elementos do HTML que são selecionados. Os items seleccionados são denominados de "selectors". Podem ser selecionados varios elementos do documento partindo da raiz, que trabalha com todo o conteúdo da página (.root) e continuando com elementos mais específicos como as tags das partes do html (h1, body, p, entre outros) e suas subdivissões (Classes e Identidades). A classe que pode ser selecionada no CSS com a expressão ".nome da classe", e a "id" usando a expressão "#Nome da id".

Geralmente costumam ser alteradas a dimensão, disposição e cores dos elementos. Para alterar a dimensão existem várias medidas como Pixels - ()px, Porcentagens - ()% (em relação ao elemento pai), área visível ou Viewport - ()vh (altura), ()vw (Comprimento). A disposição do conteúdo no CSS pode ser feita na horizontal, vertical ou espacial. A posição horizontal pode ser resolvida com "text align", já a posição vertical é mais trabalhosa. Para poder definir o alinhamento é necesário, em primeiro lugar, transformar o bloco da seção onde está o conteúdo a ser realocado em um Grid que permite trabalhar com duas dimensões: horizontal - linhas e vertical/colunas (outra opção seria a "Flexbox", mas ela é mais limitada permitindo trabalhar em uma única dimensão). Para isso é necessário mudar o display padrão (Block) para Grid: "display: grid". Após definir a disposição do display o passo seguinte é dar um comando que pode ser "align-itens" (vertical), "justify-itens" (horizontal) e "place-itens" (que trabalha tanto na horizontal quanto na vertical). Outro atributo que costuma ser alterado é a cor. Para alterar a cor existe, também, varias formas: cores básicas, números hexadecimais #(6 dígitos) (2 primeiros digitos são a quantidade de vermelho, os dois centrais a quantidade de verde, e os dois últimos a quantidade de azul) a escala crescente de cada número vai de 0 a 16 (0,1,2,..,8,9,a,b,..,e,f), valores RGBA (hexadecimal com apenas três dígitos e o A que corresponde à transparência que pode ir de 0 a 1 com intervalos sendo frações). Para testar elementos do estilo pode ser empregada a função inspecionar da pagina que esta sendo criada e de uma página que tem recursos de interesse. Podem ser alteradas as configurações de estilo com a visualização imediata das modificações feitas. Assim que é alcançado o efeito desejado (cor, forma ou tamanho), basta copiar os comandos.

Para possicionar elementos na página pode ser usada a tag "div" no html e criar subdivissões da mesma com novas divs dentro da mesma. Os elementos de forma padrão são amostrados na forma de bloco. Qualquer parte da estrutura é um bloco.

JavaScript

É uma linguagem simples que cria roteiros em uma página. Atribui interatividade para elementos. Com ela é possível manipular os objetos de uma página e criar novos elementos a partir de uma interação do usuário. Ela é interpretada e orientada a objetos utilizado funções de "primeira classe".

Funções de primeira classe significa que a linguagem permite que funções sejam argumentos de outras funções, e que seus resultados passem a ser valores para outras funções. O resultado final são variaveis ou dados alocados em estruturas de dados. O nome atribuido à função não possuem nenhum status especial, são considerados apenas cp,p variaveis com um tipo de função. Para implementar funções podem ocorrer empecilhos quando são inseridas variaveis que não são locais da pagina que está sendo desenvolvida. E suma, funções de primeira classe permitem tres coisas: 1) Que as funções possam agir como vairaves; 2) Que as funções possam servir como argumentos para outras funções; e 3) Que funções sejam o retorno de outras funções.

Para entender Javascript é necessário conhecer uma série de conceitos relacionados com funções. Alguns deles são:

  • Declaração de função (Function statement): É uma função que foi definida e pode ser empregada futuramente.
  • Expressão da função (Function expression): Após a criação de uma função e atribuida a uma variável para poder ser denominada pelo nome dessa variavel. Se não for definida não pode ser executada.
  • Função anonima (Anonymous function): É uma função na qual não há necessidade de definir um nome para a mesma, ela pode ser atribudia a uma variável.
  • Nome da função expressão (Named function expression): Função comum com um nome atribuido a uma variável
  • Parametros:
  • Argumentos:
  • Funções de primeira classe:
  • Função de seta ou direção (Arrow functions).
  • Escopo e cadeia de escopo (Scope e Scopechain): REfere-se à acessibilidade e visibilidade de variaveis. Trata sobre quais partes do progrma tem acesso a uma variavel ou em qual lugar essa variavel pode ser observada. Ao ser atribuido um escopo o programa conta com uma medida de segurança por permitir que as variaveis possam ser acessadas a partir de apenas uma parte do programa. Evita modificações não esperadas para as vaiareis e outras partes do programa. O escopo evita que ocorra a colição do espaço do nome. Isto é, podem ser usadas as mesmas variáveis para varios escopos. Os escopos podem ser divididos em três tipos: 1) Escopo global (qulaquer variavel que não estiver dentro de uma função ou bloco, esta dentro do escopo global. As variaveis podem ser acessadas em qualquer parte do programa); 2) Escopo local ou escopo de função (variaveis s~]ap declaradas dentro da fnção. Elas só podem ser acessadas de dentro da função e não podem ser acessadas fora do código); 3) Escopo de bloco (com os tipos de variaveis let e const é possivel que as mesmas possam ser atribuidas a uma exressão dentro de chavões proximas "{...}"). Outros conceitos são o aninhamento do escopo (que é quando um escopo é inserido dentro de um outro) permintindo a junção dos 3 tipos de escopos (global, foco e bloco) e o escopo léxico (também conhecido como escopo estático) significa que o escopo será determinado pelo tempo de interpretação e compilação dod dados no lugar de ocorrer pela ordem de execução (função "printNumber ()" e "console.log(number)"). Na cadeia de escopo quando uma variavel é usada no JavaScipt, o motor vai tentar localizar as variaeis dentro do escopo. Caso não localizr a variavel ira tentar localiza-la em outro escopo mais externo e assim por diante até localizar a variavel em um escopo global.
